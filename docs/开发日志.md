# 开发日志

## 2026-03-16

### 本地原生 CLI 入口

- `scripts/start-chat.sh` 不再默认进入本地统一聊天壳
- 本地启动改成：
  - 先从共享项目库选择项目
  - 再直接进入原生 `claude` / `codex` CLI
- 新增全局入口：
  - `ash-chat`
- 本地进入原生 CLI 时会注入：
  - `ASH_ACTIVE_WORKSPACE`
  - `CCB_WORK_DIR`
  - `CCB_RUN_DIR`

### 本地项目选择器收口

- 本地项目选择器不再把任意输入当成新 workspace
- 现在只能：
  - 选择现有项目编号
  - 输入现有项目名
  - 输入 `temporary`
- 修复误输入 `project` 导致本地库里产生垃圾 workspace 的问题
- 本地启动器现在优先读取共享项目库，不再只看本地 `workspaces`
- 删除误建的本地 workspace：`project`

### 本地与远程命令统一

- 新增本地统一聊天入口 `local-chat`
- 新增启动脚本 `scripts/start-chat.sh`
- 本地 CLI 现在和 Telegram / 飞书共用同一套命令：
  - `/projects`
  - `/use <workspace>`
  - `/where`
  - `/write`
  - `/execute`
  - `/status`
  - `/sessions`
  - `/worker`
  - `/tasks`

### 项目与临时模式分离

- 明确“项目”是正式长期对话和任务管理的唯一基础
- 未绑定项目时，不再自动创建默认 workspace
- 未绑定项目的短消息只进入 `ephemeral`
- 未绑定项目的长消息进入 `Temporary Swarm Mode`
- `Temporary Swarm Mode` 支持自动选择起手 agent：
  - 讨论类默认 `claude`
  - 实现/修复/测试类优先 `codex`
- 临时模式不进入项目长期记忆

### 项目 worker 强化

- 项目模式继续保留 `Claude -> Codex -> Claude` 的正式工作流
- 新增大任务自动规划：
  - 复杂任务会先进入 `planning`
  - Claude 生成 `execution_plan`
  - `execution_plan` 包含：
    - `complexity`
    - `needs_subagents`
    - `suggested_subagents`
    - `planner_output`

### sub-agent 真派发

- `suggested_subagents` 不再只是建议
- worker 会真实派发 sub-agent
- 新增结构化交接：
  - `subagent_packet`
  - `subagent_result`
- sub-agent 结果会回收并并入主 `execution_packet`

### 监控视图

- 新增 `/worker`
  - 查看 phase
  - 当前 task
  - Claude / Codex session
  - 最近 handoff
  - sub-agent 运行次数
- 新增 `/tasks`
  - 查看当前 workspace 的最近任务列表

## 2026-03-15

### 项目会话库与命名清理

- 共享项目库项目名从哈希后缀改为可读名字：
  - `agent-swarm-hub`
  - `claude`
  - `codex-history`
  - `sunxiangrong`
- 补充项目 `profile`
- 项目 summary 从单层启发式摘要升级为：
  - 长期 `profile`
  - 动态 `summary`

### 项目画像可手动维护

- 新增项目画像脚本：
  - `set_project_profile.py`
  - `show_project_profile.py`
- `trim_context.py` 会优先输出：
  - `project_profile`
  - `project_summary`
- `agent-swarm-hub` 的 `/where`、`/projects` 和 prompt 注入都开始使用手动维护的长期画像

### 对话生命周期优化

- 新增 `ephemeral_messages`
- 短对话默认进入临时缓冲，不进入正式项目记忆
- 临时消息有：
  - TTL
  - 条数上限
- 新增 `/sessions` 方便查看正式消息和临时消息数量

### 项目上下文与共享索引接入

- `agent-swarm-hub` 开始读取共享项目会话库
- workspace path 对上共享项目库后，会自动带出：
  - 项目 summary
  - provider session 统计
  - active session 数量

## 2026-03-14

### 远程入口稳定化

- Telegram 改为常驻轮询
- Telegram 网络异常自动重试
- 飞书 WebSocket 自动重连
- 修复 Telegram `reply_to_message_id` 失效导致进程崩溃的问题

### 项目级 worker 初版

- 引入 workspace / workspace_session / task 持久化
- 引入 phase 状态机：
  - `discussion`
  - `ready_for_execution`
  - `executing`
  - `reviewing`
  - `reported`
- 引入 Claude / Codex 双 session：
  - `claude_session_id`
  - `codex_session_id`
- 引入双 agent 独立记忆流
- 引入结构化 handoff：
  - `discussion_brief`
  - `execution_packet`
  - `review_verdict`

### ccb 接入执行链

- 在执行层注入：
  - `CCB_SESSION_ID`
  - `CCB_WORK_DIR`
  - `CCB_RUN_DIR`
- 让 `worker_session` 可以通过 `ask/askd + ccb` 路由到底层长期会话

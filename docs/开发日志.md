# 开发日志

## 2026-03-17

### 本地入口重新收口为 swarm shell

- `ash-chat` / `scripts/start-chat.sh` 不再默认进入原生 `claude` / `codex` CLI
- 现在行为改为：
  - 先从共享项目库选择项目
  - 进入项目后直接进入本地 swarm shell
  - 本地 swarm shell 与远程聊天命令层保持一致
- 原生 `claude` / `codex` CLI 继续保留，但不再是 `ash-chat` 默认主入口

### 项目列表与 temporary 模式优化

- `/projects` 改成更适合聊天查看的项目列表：
  - 项目名
  - `Mode`
  - `Path`
  - `Profile`
  - `Focus`
- `temporary` 现在成为显式可选模式
- `/use temporary` 会清掉当前 chat 的项目绑定和临时缓冲
- 切离 `temporary` 时也会清空临时上下文，避免污染正式项目

### 子智能体语义收紧

- 主 `Codex` 负责：
  - 实现
  - 执行
  - 自校验
- `sub-agent` 的定位不再是模拟组织架构，而是：
  - 按需拉起的上下文隔离单元
- `planning`、`execution_packet`、`subagent_packet` 已统一按这个语义收紧
- 只读任务会自动转成只检查/只汇报，不再给 `Codex` 泛化“实现并验证”指令

### 小型分层 md 与按需加载

- 新增并接入小型分层文档：
  - `project-entry.md`
  - `worker-flow.md`
  - `codex-execution.md`
  - `subagent-policy.md`
  - `remote-shell.md`
- 这些文档已不再只是静态说明，而是按 `phase + agent + mode` 进入 prompt 组装
- 做到了渐进式披露，而不是把全部规则一次塞进上下文

### ccb / askd bridge 稳定性修复

- 修复 `ccb` 对 `codex` 的启动目录传递：
  - `codex -C <project_root>`
- 增强 Claude / Codex 初始 trust prompt 处理
- `ask` 现在可以按当前 cwd 自动推断 project-scoped `askd`
- `ccb-ping claude` / `ccb-ping codex` 都能正常变绿

### 交互确认上浮

- Claude 的执行型确认页开始上浮到统一命令层
- 包括新版 `go ahead` 这类确认提示
- 在：
  - 本地 swarm shell
  - 远程聊天
  中都会返回可见提示
- 用户可通过 `/confirm` 继续
- 自动 probe / 自动流水线则允许自动 `/confirm`

### 真实 bridge 联调结果

- Claude planning 现在可以靠：
  - 显式 `CCB_DONE`
  - 或 idle 收口
  正常继续往后走
- 真实 fresh `ccb-swarm` 下，新的主要阻塞点不是任务流，而是 `Codex` 认证页
- 这类启动级认证现在也会被识别并上浮为：
  - `Authentication Required`
- 因此系统已从“静默卡死”升级为“明确告警并等待处理”
- 当前离真实全自动多 agent 闭环还差一步：
  - 先完成一次 Codex 登录
  - 再重跑 end-to-end swarm probe

## 2026-03-16

### 本地原生 CLI 入口

- `scripts/start-chat.sh` 不再默认进入本地统一聊天壳
- 本地启动改成：
  - 先从共享项目库选择项目
  - 再直接进入原生 `claude` / `codex` CLI
- 新增全局入口：
  - `ash-chat`
- 本地进入原生 CLI 时会注入：
  - `ASH_ACTIVE_WORKSPACE`
  - `CCB_WORK_DIR`
  - `CCB_RUN_DIR`

### 本地项目选择器收口

- 本地项目选择器不再把任意输入当成新 workspace
- 现在只能：
  - 选择现有项目编号
  - 输入现有项目名
  - 输入 `temporary`
- 修复误输入 `project` 导致本地库里产生垃圾 workspace 的问题
- 本地启动器现在优先读取共享项目库，不再只看本地 `workspaces`
- 删除误建的本地 workspace：`project`

### 本地与远程命令统一

- 新增本地统一聊天入口 `local-chat`
- 新增启动脚本 `scripts/start-chat.sh`
- 本地 CLI 现在和 Telegram / 飞书共用同一套命令：
  - `/projects`
  - `/use <workspace>`
  - `/where`
  - `/write`
  - `/execute`
  - `/status`
  - `/sessions`
  - `/worker`
  - `/tasks`

### 项目与临时模式分离

- 明确“项目”是正式长期对话和任务管理的唯一基础
- 未绑定项目时，不再自动创建默认 workspace
- 未绑定项目的短消息只进入 `ephemeral`
- 未绑定项目的长消息进入 `Temporary Swarm Mode`
- `Temporary Swarm Mode` 支持自动选择起手 agent：
  - 讨论类默认 `claude`
  - 实现/修复/测试类优先 `codex`
- 临时模式不进入项目长期记忆

### 项目 worker 强化

- 项目模式继续保留 `Claude -> Codex -> Claude` 的正式工作流
- 新增大任务自动规划：
  - 复杂任务会先进入 `planning`
  - Claude 生成 `execution_plan`
  - `execution_plan` 包含：
    - `complexity`
    - `needs_subagents`
    - `suggested_subagents`
    - `planner_output`

### sub-agent 真派发

- `suggested_subagents` 不再只是建议
- worker 会真实派发 sub-agent
- 新增结构化交接：
  - `subagent_packet`
  - `subagent_result`
- sub-agent 结果会回收并并入主 `execution_packet`

### 监控视图

- 新增 `/worker`
  - 查看 phase
  - 当前 task
  - Claude / Codex session
  - 最近 handoff
  - sub-agent 运行次数
- 新增 `/tasks`
  - 查看当前 workspace 的最近任务列表

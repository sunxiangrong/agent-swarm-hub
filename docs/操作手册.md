# 操作手册

## 1. 准备

使用 `cli` 环境：

```bash
conda activate cli
cd /Users/sunxiangrong/dev/cli/git/agent-swarm-hub
```

本地配置文件：

```bash
cp .env.local.example .env.local
```

建议配置：

```bash
ASH_EXECUTOR=claude
ASH_EXECUTOR_TRANSPORT=ccb
ASH_PROXY_URL=http://127.0.0.1:6789

ASH_TELEGRAM_ENABLED=true
ASH_TELEGRAM_BOT_TOKEN=你的 token

ASH_LARK_ENABLED=true
ASH_LARK_APP_ID=你的 app id
ASH_LARK_APP_SECRET=你的 secret
ASH_LARK_VERIFY_TOKEN=你的 verify token
```

## 2. 启动

Telegram：

```bash
./scripts/start-telegram.sh
```

飞书：

```bash
./scripts/start-lark.sh
```

同时启动：

```bash
./scripts/start-local.sh
```

## 3. 项目级使用方式

查看项目：

```text
/projects
```

切换项目：

```text
/use agent-swarm-hub
```

查看当前项目：

```text
/where
```

设置项目目录：

```text
/project set-path /Users/sunxiangrong/dev/cli/git/agent-swarm-hub
```

设置项目后端：

```text
/project set-backend claude
```

设置项目传输层：

```text
/project set-transport ccb
```

## 4. 任务工作流

开始一个新任务：

```text
/write 先和我讨论 Telegram 与飞书项目 worker 流程
```

继续讨论：

```text
把 Claude 负责的 review 流程再细化一下
```

进入执行：

```text
/execute 按刚才讨论的方案开始实现并跑验证
```

查看状态：

```text
/status
```

清空当前任务上下文：

```text
/new
```

## 5. 当前状态解释

`/status` 和 `/where` 会显示：

- `Workspace`
- `Executor Session`
- `Claude Session`
- `Codex Session`
- `Phase`

其中 `Phase` 含义：

- `discussion`
  当前在和 Claude 讨论
- `ready_for_execution`
  讨论已成型，等待进入执行
- `executing`
  Codex 正在执行
- `reviewing`
  Claude 正在 review Codex 结果
- `reported`
  Claude 已经给出最终汇报

## 6. ccb 在哪里起作用

当 workspace 的 `transport=ccb` 时：

- `worker_session` 会把 agent 的会话 ID 注入：
  - `CCB_SESSION_ID`
  - `CCB_WORK_DIR`
  - `CCB_RUN_DIR`
- `ask/askd` 会按 `work_dir -> ccb_project_id -> provider` 找回对应后台会话
- 这样 `claude_session_id` 和 `codex_session_id` 就能稳定映射到 CCB 里的长期会话

一句话：

`project worker` 负责调度，`ccb` 负责把 Claude/Codex agent 变成长期后台会话。

## 7. 推荐日常操作

日常最常用的顺序：

1. `/use <project>`
2. `/project set-path <path>`
3. `/project set-transport ccb`
4. `/write ...`
5. 普通文本继续讨论
6. `/execute ...`
7. `/status`

## 8. 常见问题

Telegram 没回复：

- 看启动终端是否有网络/代理错误
- 确认 `.env.local` 里的 `ASH_TELEGRAM_BOT_TOKEN`

飞书断开：

- 当前已支持自动重连
- 若长期不稳，优先检查代理是否支持 WebSocket

Claude/Codex 没走 CCB：

- 先确认 workspace 的 `transport` 是 `ccb` 或 `ask`
- 再确认本机 `ask/askd` 可用

如果你只想检查当前配置：

```bash
conda run -n cli python -m agent_swarm_hub.cli telegram-poll --print-config
conda run -n cli python -m agent_swarm_hub.cli lark-ws --print-config
```
